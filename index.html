字数统计: 1.5k | 阅读时长 ≈ 5

23号一面10分钟就完了，我以为就凉凉了。没想到在开学的第一天，3月1号下午接到了今天二面的通知。真的很感谢字节，给了我二面的机会。二面的面试官是个小姐姐，很好看很温柔，也很善于引导。整体下来，总结一下：

  • 模糊不清的东西，一定要整明白，不能含含糊糊。（ps:原型链）

  • 面试前几天，保证好正常的饮食作息，不要有太大压力。
    前几天我真的十分焦虑，大姨妈本来1号来，一直到今天刚面试完，刚来。
    没杂吃饭，没咋睡觉，感觉也不困不饿，但是一到面试的时候，加上紧张，就感觉很不在状态。下来之后复盘，很多回答都不是很满意。今天从早上8点到下午2点回宿舍，脑子都是涨的，也没胃口吃东西，想躺着睡会也睡不着。

  • 心态要放好。
    虽然是面试决定money，但是面试官并没有那么可怕，结果也并没有那么重要。焦虑没有用，反而会适得其反，导致状态不好。

  • 没有白走的路，每一步都算数。
    妍姐问我：“你还活着？” 我说：“凭一口仙气吊着”
    真的不要临时抱佛脚。

  • 失败不可怕，可怕的是连失败的勇气都没有
    不要觉得回答不上来问题，没有下次面试的通知就很丢人。面试这个东西，我感觉很奇妙。没有OC不着急，一切都只是缘分还没到，多投几回就好了。

前言：

  • Node.js是一个能够在服务端运行js的开放源代码，跨平台js运行环境。

  • 采用Google开发的v8引擎运行js代码，使用事件驱动，非阻塞，异步I/O模型等技术来提高性能，优化应用程序的传输量和规模

  • Node是事件驱动的。开发者可以在不使用线程的情况下开发出一个能够承载高并发的服务器。

  • 常用框架有Express.js，Socket.IO和Connect等。

  • Node仅仅对ES标准进行了实现，所以在Node中不包含DOM 和 BOM

  • Node中可以使用所有的内建对象
    String Number Boolean Math Date RegExp Function Object Array
    而BOM和DOM都不能使用

    但是可以使用 console 也可以使用定时器（setTimeout() setInterval()）

  • Node可以在后台来编写服务器
    Node编写服务器都是单线程的服务器

字数统计: 3.6k | 阅读时长 ≈ 13

一.HTTP篇

1.HTTP与HTTPS

  • 概念：
    http:超文本传输协议，用于从服务器传输超文本到本地浏览器的传输协议明文 端口：80
    https：http+ssl 加密 端口：443
  • 工作流程：
    • https: （包含了tcp/ip的3次握手）
      1.客户端发url给服务端，要求建立连接
      2.服务端接受请求后，将网站证书（含公钥）返回给客户端
      3.客户端产生随机值Key,key对公钥进行加密后传给服务端
      4.服务端用私钥解密key，再用key对传输内容加密后传给客户端
      5.客户端用key解密数据，获得数据
    • 加密算法：
      对称加密和非对称加密：对称加密算法比非对称加密要快，但是对称加密要求密钥对等。导致易被解获，在n结点的传输网络中要n-1个密钥，非常繁琐。非对称不存在这些问题，被广泛应用。
  • 对比
    • https不足
      1.证书问题–》费用高
      2.握手费时
      3.缓存不如http高效
